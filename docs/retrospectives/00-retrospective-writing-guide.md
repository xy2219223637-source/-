# 回顾写作指南

## 用途

当团队需要理解为什么某次交付即便完成了实现工作，仍然没有达到预期时，就使用 `docs/retrospectives/`。

## 适用场景

- 原型与实现的偏差比预期更大
- 需求来源不够充分
- PM 到开发的交接跳过了分析工作
- 同一个流程缺口反复出现
- 需要提炼一个新的可复用技能或审计提示

## 应包含内容

- 原始源输入
- 缺失了什么，或者哪里具有误导性
- 只有在实现过程中才发现了什么
- 下一次应该更早放进流程里的内容
- 应该创建什么新的技能、审计提示或工作流规则

## 文件名建议

优先使用带日期的文件名：

- `docs/retrospectives/YYYY-MM-DD-topic.md`
