# 需求索引

本目录用于实现就绪的需求综合。

不要把原始 PM 聊天、复制来的文章或原型笔记直接倒在这里。它们应该先进入 `docs/input/`。

## 规则

只有当内容已经清楚到足以驱动设计或实现时，才把它移到这里。

这样可以保持“原始输入”和“已落定的需求意图”之间的区分。

## 建议文件

- `00-requirement-synthesis-guide.md`
- `product-scope.md`
- `mvp.md`
- `feature-001-<name>.md`
- `2026-05-21-user-management.md`
- `2026-05-21-order-refund-flow.md`

建议：

- 长期存在的基线文件，如 `product-scope.md` 和 `mvp.md`，使用稳定名称
- 一次性的、与某轮工作绑定的需求综合文件，使用带日期的文件名
