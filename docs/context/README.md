# 上下文索引

## 用途

`docs/context/` 存放 AI 代理在做非平凡变更前应先阅读的简短、强制性上下文。

这个目录刻意保持精简。之所以存在，是因为藏在 `docs/references/` 下的重要项目规则很容易被 AI 代理漏读。

## 先读这些

1. `project-context.md`
2. `ai-autonomy-policy.md`
3. `codebase-map.md`
4. `source-of-truth-and-precedence.md`
5. `conventions.md`

## 这里应该放什么

- 每次 AI 会话都应先对齐的当前项目上下文
- 用于决定 AI 何时可以继续、计划、询问、研究或停止的自治规则
- 防止重复重新探索的简洁代码库路由信息
- 事实来源与 owner 文档优先级规则
- 不是单纯可选参考材料的项目级约定

## 这里不应该放什么

- 冗长的设计文档
- 一次性的分析
- 每日日志
- 执行计划
- 详细参考表

这些内容应放到各自所属的目录里，只在需要时从上下文中链接过去。
