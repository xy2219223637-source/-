# 文章索引

## 用途

`docs/articles/` 保存关于本模板方法论的面向外部的说明性写作。

这些文章不是日常执行时的操作事实来源。它们的作用，是帮助新读者理解这个模板为什么会被设计成现在这样。

## 先读这些

- `from-spec-driven-development-to-attractor-guided-engineering.md` - 面向业务应用团队，对比 AGE 与规格驱动工作流
- `attractor-before-harness-ai-large-scale-development-methodology.md` - 解释为什么稳定的收敛结构必须先于支架和护栏的的方法论文章

## 备注

- 保持这些文章具有解释性和可读性。
- 不要把必须执行的操作规则搬到这里；这些规则应放在 `AGENTS.md` 或 `docs/context/`。
- 当文章与 owner 文档不一致时，以 owner 文档为准。
